# 智能控制器使用说明书

## 1. 产品概述
本控制器为专用型数据采集与逻辑控制终端，内置标准 Modbus RTU 从站协议。支持多路传感器数据采集、物理按键状态监测，并具备多路继电器/阀门控制输出功能。系统内置非甲烷总烃（NMHC）超标自动报警逻辑，适用于环保监测与自动化控制场景。

---

## 3. 通讯接口参数

本控制器提供三个独立的 RS485/RS232 通讯端口（标识为 U1、U2、U4）。
各端口**出厂默认参数**如下：
- **通讯协议:** Modbus RTU (从站)
- **设备地址:** `20` (`0x14`)
- **波特率:** `9600`
- **数据位:** `8`
- **停止位:** `1`
- **校验位:** `无`

### 2.1 端口参数配置寄存器 (03/04/06/10 功能码)
用户可通过 Modbus 写入保持寄存器来独立修改各端口的通讯参数。**修改后需断电重启方可生效**。

| 寄存器地址(十进制) | 功能说明 | 对应端口 | 取值范围与说明 |
| :---: | :--- | :---: | :--- |
| **60** | U1 设备地址 | U1 | 1 ~ 247 |
| **61** | U1 波特率 | U1 | 0=2400, 1=4800, **2=9600**, 3=19200, 4=38400, 5=57600, 6=115200 |
| **62** | U1 数据位 | U1 | 默认 8 |
| **63** | U1 校验位 | U1 | **0=无校验**, 1=奇校验(ODD), 2=偶校验(EVEN) |
| **64** | U1 停止位 | U1 | 默认 1 |
| **65~69** | U2 通讯参数 | U2 | 同上 (顺序为：地址、波特率、数据位、校验位、停止位) |
| **70~74** | U4 通讯参数 | U4 | 同上 (顺序为：地址、波特率、数据位、校验位、停止位) |

**【通信示例】修改 U1 端口的设备地址为 25 (`0x19`)：**
> **发送报文 (HEX):** `14 06 00 3C 00 19 89 0B`
> *(解析：地址 20(14)，功能码 06，寄存器 60(00 3C)，写入值 25(00 19)，CRC 校验)*
> **返回报文 (HEX):** `14 06 00 3C 00 19 89 0B`
> *注意：设备回复成功后，请断电重启控制器使新地址生效。*

## 3. 运行状态与数据寄存器

设备所有的运行状态、输入按键、输出继电器状态均可通过 `03` 功能码（读保持寄存器）读取，方便上位机一键轮询。

### 3.1 离散状态寄存器列表 (03 功能码)
每个状态占用一个独立的 16 位寄存器，数值仅为 `0`（正常/断开）或 `1`（报警/闭合）。

| 寄存器地址 | 状态名称 | 状态值含义 |
| :---: | :--- | :--- |
| **51** | 系统维护状态 | 0 = 正常运行， 1 = 处于维护模式 |
| **52** | 综合报警状态 | 0 = 正常， 1 = 发生报警（如NMHC超标） |
| **53** | 采样泵输出状态 | 0 = 停止， 1 = 运行 |
| **54** | 标定阀输出状态 | 0 = 关闭， 1 = 开启 |
| **55** | 原位标定阀输出状态 | 0 = 关闭， 1 = 开启 |
| **56** | 采样按钮输入状态 | 0 = 未按， 1 = 按下 |
| **57** | 标定按钮输入状态 | 0 = 未按， 1 = 按下 |
| **58** | 原位标定按钮输入状态 | 0 = 未按， 1 = 按下 |

**【通信示例】读取设备的系统维护状态与综合报警状态（读取 51、52 寄存器）：**
> **发送报文 (HEX):** `14 03 00 33 00 02 36 C5`
> *(解析：读取起始寄存器 51(00 33)，读取数量 2)*
> **返回报文 (HEX):** `14 03 04 00 00 00 01 32 6F`
> *(解析：返回 4 字节数据，寄存器 51 为 0，寄存器 52 为 1，表示系统正常，但有报警发生)*

---

### 3.2 报警阈值设定 (10 功能码)
非甲烷总烃(NMHC)报警阈值为标准 IEEE754 **Float32** 浮点数格式（ABCD字节序），占用 2 个寄存器。此参数写入后**实时生效并永久保存**。

| 寄存器地址 | 参数名称 | 数据类型 | 默认值 |
| :---: | :--- | :--- | :--- |
| **30 ~ 31** | NMHC 报警阈值 | Float32 | 60.0 |

**【通信示例】将报警阈值设定为 50.0（浮点数 50.0 对应的 HEX 为 `42 48 00 00`）：**
> **发送报文 (HEX):** `14 10 00 1E 00 02 04 42 48 00 00 DA 69`
> *(解析：向寄存器 30(00 1E) 写入 2 个寄存器，数据长度 4 字节，数据为 42 48 00 00)*
> **返回报文 (HEX):** `14 10 00 1E 00 02 23 0D`

## 4. 设备控制指令 (01 / 05 功能码)

上位机可以直接通过操作 Modbus 线圈来控制设备的继电器和电磁阀。

| 线圈地址 | 控制对象 | 控制指令说明 |
| :---: | :--- | :--- |
| **0** | 采样泵 | 写 `FF 00` 开启， 写 `00 00` 关闭 |
| **1** | 标定阀 | 写 `FF 00` 开启， 写 `00 00` 关闭 |
| **2** | 原位标定阀 | 写 `FF 00` 开启， 写 `00 00` 关闭 |

**【通信示例】强制开启“采样泵”（线圈 0）：**
> **发送报文 (HEX):** `14 05 00 00 FF 00 8F C6`
> **返回报文 (HEX):** `14 05 00 00 FF 00 8F C6`

---

## 5. 自动报警控制逻辑
控制器内部运行独立的硬件保护与逻辑判定：
1. 系统持续监测内部寄存器中更新的 NMHC 浓度值。
2. 当 `当前 NMHC 浓度 > 设定的报警阈值 (寄存器 30-31)` 时，系统会：
   - 自动闭合物理报警继电器输出。
   - 将综合报警状态寄存器（地址 52）置为 `1`。
3. 当浓度回落至阈值以下时，自动断开报警继电器并清除报警标志位。
